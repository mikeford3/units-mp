Customization Points¶

template<typename Rep>
constexpr bool units::treat_as_floating_point = std::is_floating_point_v<Rep>¶

Specifies if a value of a type should be treated as a floating-point value.

This type trait should be specialized for a custom representation type to specify that values fo this type should be treated by the library as a floating-point ones which will enable implicit conversions between quantities.

Template Parameters

  • Rep: a representation type for which a type trait is defined

template<typename Rep>
struct quantity_values¶

A type trait that defines zero, one, min, and max for a representation type.

The zero, one, min, and max member functions in units::quantity forward their work to these methods. This type can be specialized if the representation Rep requires a specific implementation to return these quantity objects.

Template Parameters

  • Rep: a representation type for which a type trait is defined

template<typename T>
struct quantity_like_traits¶

Provides support for external quantity-like types.

The type trait should provide the following nested type aliases: dimension, unit, rep, and a static member function number(T) that will return the raw value of the quantity.

Usage example can be found in units/chrono.h header file.

Template Parameters

  • T: the type to provide support for

template<typename T>
struct quantity_point_like_traits¶

Provides support for external quantity point-like types.

The type trait should provide the following nested type aliases: dimension, unit, rep, and a static member function relative(T) that will return the quantity-like value of the quantity point.

Usage example can be found in units/chrono.h header file.

Template Parameters

  • T: the type to provide support for
